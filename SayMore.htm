Part I
Introduction

Note: This document is only a first draft, the polished document will be available at lingtran.net.

SayMore is a tool you use to organize video, audio, image, and various additional files with appropriate meta data. SayMore also helps you keep track of your recording progress. You can easily see which sessions are In progress, Incoming, or skipped. When you want to use your data elsewhere, SayMore allows you to export to Audacity, ELAN, FLEx, Toolbox, or YouTube. When you are ready, SayMore can help archive your project or session in IMDI format, ready to use with ARBIL or other IMDI-compatible utilities. SIL members can do the same to quickly submit a package of all the relevant files and meta data using SIL's RAMP archive-submission application.

1
Install SayMore

SayMore can be downloaded from https://software.sil.org/saymore/download/ . Pay attention to the technical requirements (at time of writing, Saymore requires Windows 7, 8, or 10, a PDF reader, and Microsoft .Net 4.6).

If you are going to be away from fast Internet, go ahead and download FFmpeg. FFMpeg is the open source tool SayMore uses to convert media files from the ones your device outputs to what you want for archiving.

Also if you are holding a workshop go ahead and download .Net 4.6 Standalone Installer and take it with you in case some participants don't have it.

After installation, open the SayMore application. You will see this screen:

Figure 1.1 First time opening of SayMore

Here you can re-open previous projects, browse for projects that might be stored on a USB device, and Create new, blank project.

5
Managing a Project

When you create a new project or even opening an existing project it is important to make sure the Project Tab elements are filled out as much as possible. This will also be the first tab open when you create a new project. The Project Tab is designed to collect metadata that concerns the entire project. These metadata and documents are critical for the discoverability and use of the archive.

5.1 About This Project

Below describes what should be placed in the fields in this window:

  • Title - The full title for the project. Initially, the project name appears here. You can edit it, such as to make it more formal (longer). This will not change the project name, or any file or folder names.
  • Description - Information about the scope and goals of the project.
  • Vernacular - The ISO-639 name of the langage used in the sessions.
  • Location/Address - The location, such as the village or town, or address where the session was recorded or originated.
  • Region - The région or sub-region that has the location or address.
  • Country - A list which is used to identify the country where the location/address and région are located.
  • Continent - A list which is used to identify the continent that has the country.
  • Contact Person - Contact information about the person or institution responsible for the project.
  • Funding Project Title - A title that matches associated project funding documentation.
  • Date Available - The date of that this project was or will be archived.
  • Rights Holder - The copyright symbol, year, organization name, and statement regarding copyright.
  • Depositor - The person responsible for depositing the resource in an archive.
5.2 Accès Protocol

Accès is how you determine who has permission to get or see your archived data. Accès Protocol is used to determine what access to assign to an item in archives. Here is where you choose the access protocol for the project.

  • Unless your organization instructs you differently:
    SIL members or others with permission to use REAP/RAMP, choose:
    • REAP
  • All others, choose one of the following:
    • AILCA
    • AILLA
    • ANLA
    • ELAR
    • TLA
    • Custom

If you choose 'Custom' you are presented with an empty box that you can type in custom access choices separated by commas. This allows you to set different recorded sessions to different access protocols.

5.3 Project Description Documents

This is where you add documents that describe the project and corpus. When the project is archived with IMDI the files here are exported to a special session named 'Project Descriptive Documents'.

5.4 Other Documents

This is where you add any other project-level documents that do not belong in Description Documents. When the project is archived with IMDI the files here are exported to a special session named 'Other Project Documents'.

7.2 Informed Consent

It is very important to remember to make note of everyone who has participated in the project, not just those making recordings. If you add someone to the project and don't have their consent, it is possible that it may prevent the work to proceed or even be distributed when finished due to questions about the legality of the body of work gathered. So before anyone contributes to the project, get their informed consent finished.

It says Informed Consent here because it is the responsibility of the researcher to inform the participants of their intellectual property rights. If the person does not understand that they have the option to refuse any use of their intellectual property, you may not, even in good faith, use the products for academic or personal purposes.

Overview of Transcription Process

As researchers on the field, you are often the most interested in your research questions. In poorly resourced languages, it is quite possible that the corpus you have collected will be a very important resource for the community, and in addition for other researchers later.

So the research serves several purposes:

  • Informed Consent(s)
  • Harvesting primary data on the field.
  • The gathering of resources found on the target langage and languages ​​around.
  • Your assumptions and your results.

SayMore will help you organize, analyze, archive, and publish data based on records.

There are six major steps to follow in SayMore by registration so that these data are usable not just for yourself, but for the researchers later.

  1. Source Recording
  2. Informed Consent
  3. Careful Speech
  4. Oral Transcription
  5. Written Transcription
  6. Written Translation

These processes are not only in order of completion, but each well-done step will make the next steps easier. So, always plan more time for successive steps. The written transcription is painful, but very useful. If you are native to the langage you are studying, you can quickly do steps 1-4, transcribe and translate (in writing) just the most useful records for the dissertation, and leave the rest of the work to others later on who receives your corpus.

If you have already created a session with audio (or video), look at the tab Stage and stages. The box next to Source Recording will be checked.

You are ready to begin entering metadata and transcribing. So SayMore will guide you through these processes.

10
Archiving Overview
10.1 Archive with RAMP (SIL)...

Archive with RAMP (SIL)... This allows you to archive using SIL's access protocol Repository for Electronic Archiving and Publishing format. It is designed to carry over all meta-data to an archive.

10.2 Archive using IMDI

Archive using IMDI.... This allows you toarchive the session as an ISLE Meta Data Initiative, it is a standard used to describe multi-media and multi-modal langage resources. When you archive with IMDI, files in Descriptive Documents are exported to a special session named 'Project Descriptive Documents' and files in Other Documents are exported to a special session named 'Other Project Documents'.

Related Topics:

  • Accès Protocol
  • Archiving overview
