Part I
Introduction

SayMore is a tool you use to organize video, audio, image, and various additional files with appropriate meta data. SayMore also helps you keep track of your recording progress. You can easily see which sessions are In progress, Incoming, or skipped. When you want to use your data elsewhere, SayMore allows you to export to Audacity, ELAN, FLEx, Toolbox, or YouTube. When you are ready, SayMore can help archive your project or session in IMDI format, ready to use with ARBIL or other IMDI-compatible utilities. SIL members can do the same to quickly submit a package of all the relevant files and meta data using SIL's RAMP archive-submission application.

1
Install SayMore

SayMore can be downloaded from https://software.sil.org/saymore/download/ . Pay attention to the technical requirements (at time of writing, Saymore requires Windows 7, 8, or 10, a PDF reader, and Microsoft .Net 4.6).

If you are going to be away from fast Internet, go ahead and download FFmpeg. FFMpeg is the open source tool SayMore uses to convert media files from the ones your device outputs to what you want for archiving.

Also if you are holding a workshop go ahead and download .Net 4.6 Standalone Installer and take it with you in case some participants don't have it.

After installation, open the SayMore application. You will see this screen:

Figure 1.1 First time opening of SayMore

Here you can re-open previous projects, browse for projects that might be stored on a USB device, and Create new, blank project.

1.1 Change user interface language

This allows you to change the Language interface.

Figure 1.2 User Interface Language Drop down

If the language you need is not listed, or you need to edit translated UI content:

  • Click the I want to localize SayMore for another language hyperlink. The Localize User Interface dialog box opens so you can use it.
  • Optionally, get a shared UI language file and a translated Access Protocol file from another user.

If not all words are changed to the selected language, just close and re-open SayMore

A Tour around SayMore

SayMore has only four menu items and three tabs. The menu Session is greyed out until you clique on the Sessions tab. This is the same for menu Person and tab People.

Figure 2.1 Menu and Tab items

The next sections will go over what you will find in the Tab layouts.

2.1 Project Tab

The project tab is where you are able to specify and keep track of what the project is about.

Figure 2.2 Project Tab window

There is a small menu to the left of the window:

Figure 2.3 Project Tab context menu

  • About This Project
    This is where you fill out the information pertaining to your project such as a Title, Description, Location, etc
  • Access Protocol
    This is where you set the access protocol used by this project. This determines who has permission to get or see your archived data.
  • Description Documents
    This is where you add files pertaining to the project and the whole body of work for a language.
  • Other Documents
    This is where you add documents that don't seem to fit anywhere else, like how the project was funded.
  • Progress
    This is where you can see the progress being made for the open project. You can even Copy, Save or Print the information found here.
2.2 Session Tab

The Sessions Tab is where you will find the information pertaining to each session as well as what was done in each session. The Session Tab window:

Figure 2.4 Example Sessions

The left side of the window is where you will see your sessions. There are columns to show the sessions ID, Title, Stages, and Status. There are other optional columns you can turn on (Date, Genre, Location) by clicking and selecting which ones you want to view.

The right upper window shows what files make up that session. The bottom right window displays information and content to be filled in depending on what you have selected in the upper right window. If you wish to add some more documentation or add the audio file for this session, click and browse to the desired file.

2.3 People Tab

The People tab is where you keep track of all your participants. Here is where you will document their biological information, contributions, and notes:

Figure 2.5 People Tab with example

Also you can add any files you need here also by clicking and browsing to the desired files.

5
Managing a Project

When you create a new project or even opening an existing project it is important to make sure the Project Tab elements are filled out as much as possible. This will also be the first tab open when you create a new project. The Project Tab is designed to collect metadata that concerns the entire project. These metadata and documents are critical for the discoverability and use of the archive.

5.1 About This Project

Below describes what should be placed in the fields in this window:

  • Title - The full title for the project. Initially, the project name appears here. You can edit it, such as to make it more formal (longer). This will not change the project name, or any file or folder names.
  • Description - Information about the scope and goals of the project.
  • Vernacular - The ISO-639 name of the language used in the sessions.
  • Location/Address - The location, such as the village or town, or address where the session was recorded or originated.
  • Region - The region or sub-region that has the location or address.
  • Country - A list which is used to identify the country where the location/address and region are located.
  • Continent - A list which is used to identify the continent that has the country.
  • Contact Person - Contact information about the person or institution responsible for the project.
  • Funding Project Title - A title that matches associated project funding documentation.
  • Date Available - The date of that this project was or will be archived.
  • Rights Holder - The copyright symbol, year, organization name, and statement regarding copyright.
  • Depositor - The person responsible for depositing the resource in an archive.

5.2 Access Protocol

Access is how you determine who has permission to get or see your archived data. Access Protocol is used to determine what access to assign to an item in archives. Here is where you choose the access protocol for the project.

  • Unless your organization instructs you differently:
    SIL members or others with permission to use REAP/RAMP, choose:
    • REAP
  • All others, choose one of the following:
    • AILCA
    • AILLA
    • ANLA
    • ELAR
    • TLA
    • Custom

If you choose 'Custom' you are presented with an empty box that you can type in custom access choices separated by commas. This allows you to set different recorded sessions to different access protocols.

5.3 Description Documents

This is where you add documents that describe the project and corpus. When the project is archived with IMDI the files here are exported to a special session named 'Project Descriptive Documents'.

5.4 Other Documents

This is where you add any other project-level documents that do not belong in Description Documents. When the project is archived with IMDI the files here are exported to a special session named 'Other Project Documents'.

5.5 Progress

Figure 5.1 Progress

Managing Sessions

If you have just created a new project, the sessions tab will be empty. Otherwise you will see all of your previous sessions and what was recorded in each session.

6.1 Create a new Session

New Allows you to create a blank new Session. You then have to manually add all the files you wish to associate with the new session. This is done on the right hand side of the Sessions Tab window.

Vous pouvez ajouter une session vide ou une session avec le contenu d'un nouvel enregistrement. De plus, vous pouvez ajouter plusieurs sessions en téléchargeant les enregistrements du jour directement sur les appareils (caméra, enregistreur audio, etc.). Dans ce cas, SayMore crée des ID pour chaque session. Chaque ID deviendra le nom de son dossier, ainsi que la première partie de la plupart des noms de fichiers dans le dossier. Ceux-ci sont automatiquement mis à jour si vous modifiez ultérieurement cette métadonnée Id.

Note:

  • Add Files... You can add files to any Session, not just a New session.

Exporting Data

9.1 Export Sessions
  1. Open the project that has the session data you want to export.
  2. On the Project menu, clique Export Sessions.
    The Export Data dialog box appears:
    Figure 9.1 Export Sessions Save Dialogue
  3. Click Save.
    The session data in the open project is exported to a file. The file is then shown in the folder.
  4. To open the file, open the folder, and then double-click the file.
    The export file opens with the program specified as the default program for the file type.
9.2 Export People

To export all of the people and associated metadata in the open project to a file, do the following steps:

  1. Open the project that has the people you want to export.
  2. On the Project menu, clique Export People.
    The Export Data dialog box appears:
    Figure 9.2 Export People Save Dialogue
  3. In the dialog box, clique the File name box, and then type a name.
    Leave the Save as type selection as CSV (Comma delimited)(*.csv).
    Also, it is recommended that you save the file in the project folder, which is the default location.
  4. Click Save.
    The people data in the open project is exported to a file. The file is then shown in the folder.
  5. To open the file, open the folder, and then double-click the file.
    The export file opens with the program specified as the default program for the file type.
